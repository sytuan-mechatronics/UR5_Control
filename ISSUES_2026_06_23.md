## 1. Camera LAN — stream stale toi 4000ms+

### Trieu chung
- `test_camera_view.py` hien thi: FPS=3.0, Reconnects=4, Stream stale 4084ms
- UR5 lien tuc gap sai toa do hoac gap lai 1 diem do frame cu

### Nguyen nhan goc re
`FULL_FRAME_REQUIRE` trong Orbbec SDK yeu cau SDK doi ca color frame va depth frame den cung timestamp.
Tren LAN (qua TL-SG1008D gigabit switch + PoE 160s), packet color va depth khong den dong thoi do jitter mang.
SDK block vo thoi han cho den khi timeout → stream stall → reconnect cycle lap lai.

Quan trong: day KHONG phai van de bang thong (TL-SG1008D la gigabit 1000Mbps).
Nguyen nhan la behavior cua SDK khi mode FULL_FRAME_REQUIRE gap jitter packet LAN.

### Fix da ap dung
**File:** `vision/femto_camera.py`

1. Skip `FULL_FRAME_REQUIRE` hoan toan khi transport la LAN:
   ```python
   if not self._is_lan_mode():
       try:
           full_frame_require = _enum_value(ob, [...], ["FULL_FRAME_REQUIRE"])
           if full_frame_require is not None:
               _call_any(self.config, [...], full_frame_require)
       except Exception as e:
           logger.warning(f"Could not set frame aggregate mode: {e}")
   ```

2. Override FPS cho LAN profile sau khi detect transport:
   ```python
   if self._is_lan_mode():
       self.fps = int(config.CAMERA_LAN_FPS)  # default 15, .env override = 10
   ```

3. Retry loop `get_aligned_frames()` tiep tuc cho den khi co depth frame (khong dung frameset khong co depth):
   ```python
   for _ in range(self._frame_retries()):
       candidate = _call_any(self.pipeline, ["wait_for_frames", ...], wait_timeout_ms)
       if not candidate:
           continue
       depth_frame_check = _call_any(candidate, ["get_depth_frame", ...])
       if depth_frame_check is not None:
           frameset = candidate
           break
       frameset = candidate  # giu lai non-None cuoi neu depth van miss
   ```

**File:** `test_camera_view.py` — ap dung cung logic skip FULL_FRAME_REQUIRE va LAN FPS override.

## 2. Stale frame buffer dan den sai toa do pick

### Trieu chung
Robot di chuyen den SCAN_POSE, goi `get_aligned_frames()`, nhung nhan frame tu luc truoc khi robot di chuyen.
TCP pose doc tai thoi diem hien tai nhung frame la cu → transform ra toa do sai → gap sai vi tri.

### Nguyen nhan goc re
SDK LAN accumulate frames lien tuc trong pipeline buffer. Khi goi `wait_for_frames()`, co the tra ve frame da buffer tu 1-4 giay truoc.

### Fix da ap dung
**File:** `vision/femto_camera.py` — them method `flush_stale_frames()`:
```python
def flush_stale_frames(self, n: int = 2) -> None:
    """Drain buffered frames on LAN transport before a real capture."""
    if not self.pipeline or not self._is_lan_mode():
        return
    flush_timeout_ms = min(self._wait_timeout_ms(), 150)
    for _ in range(n):
        try:
            _call_any(self.pipeline, ["wait_for_frames", "waitForFrames"], flush_timeout_ms)
        except Exception:
            break
```

**File:** `core/pick_place.py` — goi flush truoc moi lan chup thuc su:
```python
self.camera.flush_stale_frames(n=config.CAMERA_LAN_WARMUP_FRAMES)
rgb, depth, cam_ts = self.camera.get_frames_with_timestamp()
tcp_pose_at_capture, rtde_ts = self.rtde.get_tcp_pose_with_timestamp()
```
Ap dung ca Phase 2 (line 333) va Phase 3 (line 650).

## 3. clamp_pick_z_sequence — min_descent khong khop validation

### Trieu chung
Motion validation `_validate_pick_motion` that bai voi loi "descent span too small" du geometry co ve hop le.

### Nguyen nhan goc re
`clamp_pick_z_sequence` co default `min_descent_mm=5.0` (5mm).
Nhung `_validate_pick_motion` kiem tra `descent_span >= PICK_MIN_DESCENT_M = 0.02` (20mm).
Clamp co the tra ve approach_z chi 5mm tren final_z → validation that bai 20mm check.

### Fix da ap dung
**File:** `core/pick_place.py` — ca 2 call site (Phase 2 line 442, Phase 3 line 777):

```python
approach_z, final_z, _ = clamp_pick_z_sequence(
```

## 4. Phase 3 — vong lap gioi han, gap lap lai o cuoi khi grip fail, khong loai tru vi tri da gap

### Trieu chung
- Vong lap `for attempt in range(parts_found)` gioi han so lan pick bang so phat hien ban dau.
- Neu grip fail, phan co the bi bo qua.
- Scan tiep theo van chon vi tri da gap thanh cong (slot trong).
- Khi phat hien het phan (detect=0) khong tu dong thoat dung cach.

### Nguyen nhan goc re
Logic Phase 3 cu su dung mang index tu scan dau tien va khong co co che loai tru hay rescan.

### Fix da ap dung
**File:** `core/pick_place.py` — viet lai hoan toan Phase 3 (lines 633–937):

**a) Vong lap until detect=0:**
```python
for pick_cycle in range(config.MAX_PICK_CYCLES):  # safety ceiling = 20
    flush + capture + detect
    if not detections:
        break  # khay rong
    
    # Loai tru vi tri da gap thanh cong

        fresh = [det for det in detections if not any(
            math.hypot(det.center[0]-pu, det.center[1]-pv) < excl_radius_px
            for pu, pv in picked_uvs)]
        detections = fresh
    if not detections:
        break  # tat ca detection = da gap het
    ...
```

**b) Quick retry in-place (3 lan):**

**c) Khi grip fail het retry — rescan voi toa do moi:**
